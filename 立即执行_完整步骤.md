**操作**：

1. 打开浏览器，访问 [Supabase Dashboard](https://supabase.com/dashboard)

4. 点击 **New query**
5. 复制 `OvertimeIndexApp/setup_auto_daily_archive.sql` 的**全部内容**

7. 点击右下角 **Run** 按钮（或按 Ctrl+Enter）

**预期结果**：


如果看到 "pg_cron 扩展未启用" 的提示，这是正常的，我们会用 GitHub Actions 代替。

## 完成后的自动化

从现在开始，每天早上 **6:00 北京时间**（22:00 UTC），GitHub Actions 会自动：

1. 调用 Supabase 的 `archive_daily_data()` 函数
2. 归档前一天的数据到 `daily_history` 表
3. 应用中的7个圆点会自动显示正确的颜色
4. **无需人工干预**

1. 检查 GitHub Secrets 是否配置正确：
   - 访问：`https://github.com/zhangmengkang151286-prog/overtimework/settings/secrets/actions`
   - 确认有 `SUPABASE_URL` 和 `SUPABASE_ANON_KEY`

2. 查看 Actions 日志中的错误信息

### 问题4：应用中圆点还是显示0

1. 在 Supabase SQL Editor 中执行：
   ```sql
   SELECT * FROM daily_history ORDER BY date DESC LIMIT 7;
   ```
   确认有数据

2. 在 Supabase SQL Editor 中执行：
   ```sql
   SELECT * FROM get_daily_status(7);
   ```
   确认函数返回正确

3. 重新加载应用（摇一摇 → Reload）

4. 检查应用日志（Metro 控制台）
