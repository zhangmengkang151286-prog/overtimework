# 2月5号数据缺失 - 永久解决方案

## 🔍 问题分析

你的 7个圆点（历史状态指示器）又缺少 2月5号的数据了。

**根本原因：**
- 7个圆点显示的数据来自 `daily_history` 表
- 这个表需要每天通过 `archive_daily_data()` 函数归档
- 目前**只有 GitHub Actions** 在做归档（每天 06:00 北京时间）
- GitHub Actions 可能失败（网络问题、API限流等）
- **没有备份机制**，一旦失败就会缺数据

## ✅ 解决方案（5分钟搞定）

### 第1步：诊断问题（看看到底缺什么数据）

在 Supabase SQL Editor 中执行：

**文件：** `OvertimeIndexApp/diagnose_feb5_simple.sql`

这会显示：
- 哪些日期有归档数据（`daily_history` 表）
- 哪些日期有原始数据（`status_records` 表）

---

### 第2步：修复 2月5日数据

在 Supabase SQL Editor 中执行：

**文件：** `OvertimeIndexApp/fix_feb5_daily_archive.sql`

```sql
SELECT * FROM archive_daily_data('2026-02-05'::DATE);
```

执行后，刷新应用，2月5日的圆点应该就显示出来了。

---

### 第3步：配置 Supabase Cron（永久解决）

在 Supabase SQL Editor 中执行：

**文件：** `OvertimeIndexApp/setup_workday_archive_cron_beijing.sql`

这个脚本会：
1. 创建数据库级别的定时任务
2. 每天 **00:00 北京时间** 自动归档前一天的数据
3. 在数据库内部执行，零延迟，超可靠

执行后，你会看到：

```
任务ID | 任务名称        | 执行计划      | 是否激活
2      | daily-archive  | 0 16 * * *   | true
```

（`0 16 * * *` = 每天 16:00 UTC = 00:00 北京时间）

---

### 第4步：验证配置

在 Supabase SQL Editor 中执行：

```sql
SELECT 
  jobid as "任务ID",
  jobname as "任务名称",
  schedule as "执行计划",
  active as "是否激活"
FROM cron.job
WHERE jobname IN ('daily-archive', 'hourly-snapshot');
```

你应该看到 **2个任务**：

```
任务ID | 任务名称          | 执行计划      | 是否激活
1      | hourly-snapshot  | 0 * * * *    | true    ← 每小时快照
2      | daily-archive    | 0 16 * * *   | true    ← 每日归档（新）
```

---

## 🔒 双重保障

配置完成后，你有 **2套归档机制**：

### 主力：Supabase Cron
- ⏰ 每天 00:00 北京时间
- 📍 数据库内部执行
- ⚡ 零延迟（< 100ms）
- 🛡️ 超可靠

### 备份：GitHub Actions
- ⏰ 每天 06:00 北京时间
- 📍 GitHub 服务器执行
- ⚡ 1-5分钟延迟
- 🛡️ 可靠

**即使一个失败，另一个也能保证数据归档成功！**

## 📊 以后怎么检查

### 查看最近7天的归档数据

```sql
SELECT 
  date as "日期",
  participant_count as "参与人数",
  overtime_count as "加班人数",
  on_time_count as "准点下班人数",
  TO_CHAR(created_at AT TIME ZONE 'Asia/Shanghai', 'YYYY-MM-DD HH24:MI:SS') as "创建时间"
FROM daily_history
WHERE date >= CURRENT_DATE - INTERVAL '7 days'
ORDER BY date DESC;
```

### 查看 Cron 执行历史

```sql
SELECT 
  status as "状态",
  return_message as "返回信息",
  start_time AT TIME ZONE 'Asia/Shanghai' as "开始时间",
  end_time AT TIME ZONE 'Asia/Shanghai' as "结束时间"
FROM cron.job_run_details
WHERE jobid = (SELECT jobid FROM cron.job WHERE jobname = 'daily-archive')
ORDER BY start_time DESC
LIMIT 10;
```
