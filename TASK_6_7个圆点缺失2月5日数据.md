# Task 6: 修复7个圆点缺失2月5日和2月6日数据

## 📊 问题分析

### 当前状态（2026年2月6日）

**7个圆点显示：**
```
⚪ ⚪ ⚪ ⚪ 🟢 🔴 ⚪
1/31 2/1 2/2 2/3 2/4 1/30 2/5
```

**问题：**
- ❌ 2月5日（今天-1天）：显示白色圆点（无数据）
- ❌ 2月6日（今天）：显示白色圆点（无数据）
- ✅ 2月4日：显示绿色圆点（有数据）
- ✅ 1月30日：显示红色圆点（有数据）

### 诊断结果

**原始数据 (status_records)：**
```sql
-- 2月5日：有数据
日期       | 参与人数 | 加班人数 | 准点人数
2026-02-05 | 1        | 14       | 5

-- 2月6日：有数据
日期       | 参与人数 | 加班人数 | 准点人数
2026-02-06 | 1        | 2        | 2
```

**归档数据 (daily_history)：**
```sql
-- 2月5日：无数据 ❌
-- 2月6日：无数据 ❌
-- 2月4日：有数据 ✅
-- 1月30日：有数据 ✅
```

**结论：** `status_records` 表有数据，但 `daily_history` 表没有数据！

### 根本原因

**APP 读取的是 `daily_history` 表，不是 `status_records` 表！**

```typescript
// src/components/HistoricalStatusIndicator.tsx
// 数据来源：daily_history 表
const displayStatus = sortedStatus.slice(-7);
```

**数据流向：**
```
status_records (原始数据)
    ↓ 每天 06:00 自动归档
daily_history (归档数据)
    ↓ APP 读取
7个圆点显示
```

**问题：** 2月5日和2月6日的数据还没有归档到 `daily_history` 表！

---

## 🐛 发现的严重BUG

### 2月6日数据异常

**原始数据：**
```
用户ID | 日期       | 是否加班 | 提交时间
...001 | 2026-02-06 | false    | 17:02:26  ← 第1次提交
...001 | 2026-02-06 | true     | 17:02:32  ← 第2次提交
...001 | 2026-02-06 | false    | 18:01:18  ← 第3次提交
...001 | 2026-02-06 | true     | 18:01:23  ← 第4次提交（最后一次）
```

**归档数据（错误）：**
```
日期       | 参与人数 | 加班人数 | 准点人数 | 圆点颜色
2026-02-06 | 1        | 1        | 1        | 🟡 黄色
```

**正确结果应该是：**
```
日期       | 参与人数 | 加班人数 | 准点人数 | 圆点颜色
2026-02-06 | 1        | 1        | 0        | 🔴 红色
```

**原因：** 归档函数统计了所有4次提交（2次加班 + 2次准点），而不是只统计最后一次提交（加班）！

---

## ✅ 解决方案

### 方案概述

1. **修复归档函数** - 只统计每个用户的最后一次提交
2. **删除测试账户** - 避免无限次提交的问题
3. **创建10个普通用户** - 真实的用户数据
4. **添加 UNIQUE 约束** - 每个用户每天只能提交一次
5. **生成测试数据** - 最近7天的真实数据
6. **自动归档** - 确保 `daily_history` 表有数据

### 执行步骤

#### 第1步：修复归档函数 ⚠️ **必须先执行！**

**文件：** `OvertimeIndexApp/fix_archive_function_logic.sql`

**修复内容：**
```sql
-- 修复前（错误）：统计所有记录
SELECT COUNT(*) FILTER (WHERE is_overtime = true)
FROM status_records
WHERE date = target_date;

-- 修复后（正确）：只统计最后一次提交
WITH latest_submissions AS (
  SELECT DISTINCT ON (user_id)
    user_id, is_overtime
  FROM status_records
  WHERE date = target_date
  ORDER BY user_id, submitted_at DESC  -- 每个用户取最后一次
)
SELECT COUNT(*) FILTER (WHERE is_overtime = true)
FROM latest_submissions;
```
