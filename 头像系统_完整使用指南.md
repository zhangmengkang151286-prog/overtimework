## 🎨 SVG 文件要求

### 推荐规格
- **尺寸**：100x100 或 200x200 像素
- **格式**：SVG（矢量图）
- **文件大小**：每个文件 < 20KB
- **viewBox**：`viewBox="0 0 100 100"` 或类似正方形比例
- **颜色**：使用纯色或渐变，避免复杂滤镜

### 示例 SVG 结构

查看 `assets/avatars/avatar_01.svg` 了解基本结构：
- 背景圆形
- 脸部
- 头发
- 五官
- 衣服

### 优化 SVG

如果 SVG 文件过大或包含不必要的元素，使用在线工具优化：

**SVGOMG**（推荐）：
1. 访问：https://jakearchibald.github.io/svgomg/
2. 上传你的 SVG 文件
3. 调整优化选项
4. 下载优化后的文件

**命令行工具**：
```bash
npm install -g svgo
svgo assets/avatars/*.svg
```

## 🐛 故障排除

### 问题 1：SVG 不显示

**症状**：头像区域空白或显示错误

**解决方案**：
1. 确认 SVG 文件存在于 `assets/avatars/` 目录
2. 确认文件命名正确（`avatar_01.svg` 到 `avatar_20.svg`）
3. 清除缓存：`npx expo start --clear`
4. 重启 Metro bundler

### 问题 2：TypeScript 报错

**症状**：导入 SVG 时 TypeScript 报错

**解决方案**：
1. 确认 `src/types/svg.d.ts` 文件存在
2. 重启 VS Code 或 TypeScript 服务器
3. 运行 `npx tsc --noEmit` 检查类型错误

### 问题 3：SVG 显示异常（变形、颜色错误）

**症状**：SVG 显示但样式不对

**解决方案**：
1. 检查 SVG 的 `viewBox` 属性（应该是正方形，如 `0 0 100 100`）
2. 使用 SVGOMG 优化 SVG
3. 确认 SVG 不包含不支持的特性：
   - ❌ 外部样式表
   - ❌ JavaScript
   - ❌ 复杂滤镜
   - ❌ 动画
   - ✅ 基本形状（circle, rect, path）
   - ✅ 纯色填充
   - ✅ 渐变

### 问题 4：Metro bundler 报错

**症状**：启动时报错 "Unable to resolve module"

**解决方案**：
1. 确认 `metro.config.js` 已更新（已完成）
2. 删除 `node_modules` 和 `package-lock.json`
3. 重新安装：
   ```bash
   npm install
   npx expo start --clear
   ```
