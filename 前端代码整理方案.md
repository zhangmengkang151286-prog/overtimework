### ❌ 根目录混乱（严重问题）

**问题统计：**
- 📄 **200+ 个文档文件**（MD 文件）
- 📝 **80+ 个 SQL 脚本**
- 🧪 **10+ 个测试/备份 App 文件**
- 🔧 **多个重复的配置脚本**

**主要问题：**
1. **文档爆炸**：每次修复、调试都生成新文档
2. **SQL 脚本散乱**：数据库脚本没有统一管理
3. **临时文件未清理**：测试文件、备份文件堆积
4. **中英文混杂**：文档命名不统一

## 🎯 整理方案

### 方案 A：渐进式整理（推荐）⭐

**优点**：
- 风险低，不影响开发
- 可以逐步进行
- 保留重要历史记录

**步骤：**

#### 1. 创建文档归档目录（5分钟）

```bash
cd OvertimeIndexApp

# 创建归档目录结构
mkdir -p docs/archive/{fixes,features,database,guides}
mkdir -p scripts/{database,setup,maintenance}
mkdir -p backup
```

#### 2. 整理 SQL 脚本（10分钟）

```bash
# 移动数据库脚本
move *.sql scripts/database/

# 按功能分类
cd scripts/database
mkdir -p {setup,fixes,migrations,tests}

# 手动分类（或使用脚本）
move setup_*.sql setup/
move fix_*.sql fixes/
move diagnose_*.sql tests/
move insert_*.sql tests/
```

#### 3. 归档修复文档（15分钟）

```bash
cd OvertimeIndexApp

# 归档屏幕冻结修复文档（15个版本）
move SCREEN_FREEZE_FIX_*.md docs/archive/fixes/

# 归档任务完成文档
move TASK_*_SUMMARY.md docs/archive/features/
move TASK_*_COMPLETE*.md docs/archive/features/

# 归档数据库修复文档
move FIX_*.md docs/archive/fixes/
move *_FIX*.md docs/archive/fixes/

# 归档快速修复文档
move 快速修复_*.md docs/archive/fixes/
move 立即执行_*.md docs/archive/fixes/
```

#### 4. 整理指南文档（10分钟）

```bash
# 保留在根目录的核心文档
# - README.md
# - BUILD_GUIDE.md
# - RUN_APP_GUIDE.md
# - PRODUCTION_READY_GUIDE.md

# 移动其他指南到 docs/guides/
move *_GUIDE.md docs/guides/
move *指南.md docs/guides/
move *_SETUP.md docs/guides/
```

#### 5. 清理临时文件（5分钟）

```bash
# 移动测试 App 文件到 backup/
move App.test*.tsx backup/
move App.*.tsx backup/
move test-*.js backup/
move verify-*.js backup/

# 删除重复的清除缓存脚本（保留最新的）
# 保留：fix-glassmorphism.bat, clear-cache.bat
# 删除：其他重复脚本
```

#### 6. 创建文档索引（10分钟）

创建 `docs/INDEX.md`：

```markdown

## 📚 核心文档（根目录）

## 🗄️ 数据库脚本（scripts/database/）
- setup/ - 初始化脚本
- fixes/ - 修复脚本
- migrations/ - 迁移脚本
- tests/ - 测试数据脚本

## 📦 归档（docs/archive/）
- fixes/ - 历史修复记录
```

## 🎯 维护建议

### 1. 文档命名规范

**新文档应该放在：**
- 修复记录 → `docs/archive/fixes/`
- 功能开发 → `docs/archive/features/`
- 开发指南 → `docs/guides/`
- 设计文档 → `docs/design/`

**命名格式：**
- 日期前缀：`2026-02-13_功能名称.md`
- 版本号：`功能名称_v1.md`, `功能名称_v2.md`
- 避免：`FINAL`, `COMPLETE`, `V15` 等混乱命名

### 2. SQL 脚本管理

**新脚本应该放在：**
- 初始化 → `scripts/database/setup/`
- 修复 → `scripts/database/fixes/`
- 迁移 → `scripts/database/migrations/`
- 测试数据 → `scripts/database/tests/`

### 3. 定期清理

**每月清理一次：**
- 删除 3 个月前的临时文件
- 归档已完成的功能文档
- 合并重复的指南文档
