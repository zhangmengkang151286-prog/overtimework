## ❓ 常见问题

### Q1: 为什么需要清除缓存？

**A:** Metro bundler 会缓存模块解析结果。新安装的依赖（expo-blur、expo-linear-gradient）不会自动更新到缓存中，必须使用 `--clear` 参数强制清除。

### Q2: --clear 参数做了什么？

**A:** 清除以下缓存：
- 模块解析缓存
- 转换缓存
- 依赖图缓存
- Babel 编译缓存

### Q3: 如果还是报错怎么办？

**A:** 尝试完全重新安装（最后手段）：

```cmd
cd OvertimeIndexApp

# 1. 停止所有 Node 进程
taskkill /F /IM node.exe

# 2. 删除 node_modules 和 lock 文件
rmdir /s /q node_modules
del package-lock.json

# 3. 重新安装
npm install

# 4. 清除缓存启动
npx expo start --clear
```

### Q4: 为什么要停止 Node 进程？

**A:** 确保没有进程占用缓存文件，避免删除失败。Windows 系统中，正在运行的进程会锁定文件。
