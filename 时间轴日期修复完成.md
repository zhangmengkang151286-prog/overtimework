# 时间轴日期问题修复完成

## 🐛 问题描述

1. **时间轴右侧显示 "05:59" 应该显示 "次日05:59"** ✅ 已修复
2. **拖动时间轴时数据不更新** ✅ 已修复

## 🔍 根本原因

### 问题 1：时区转换问题

日志显示：
```
workdayStart: "2026-01-30T22:00:00.000Z"  // UTC 时间
```

这是 UTC 时间，转换成北京时间（UTC+8）是 `2026-01-31 06:00:00`。

当拖动时间轴到 7:00 时：
- `TimeAxis` 组件使用 `minTime`（UTC 2026-01-30T22:00:00.000Z）
- 加上 1 小时偏移量 = UTC 2026-01-30T23:00:00.000Z
- 传递给 `getSnapshot` 时，使用 `.toISOString().split('T')[0]` 提取日期
- 结果是 `2026-01-30`（错误！应该是 2026-01-31）

### 问题 2："次日05:59" 显示问题

`TimeAxis.tsx` 的 `formatTime` 函数已经添加了"次日"前缀判断，但可能没有生效。

## ✅ 修复方案

### 修复 1：`TrendPage.tsx` - 使用本地时间创建 Date 对象

**修改前**：
```typescript
const workdayStart = new Date(now);
if (currentHour < 6) {
  workdayStart.setDate(workdayStart.getDate() - 1);
}
workdayStart.setHours(6, 0, 0, 0);
```

**修改后**：
```typescript
let workdayStart: Date;
if (currentHour < 6) {
  workdayStart = new Date(now.getFullYear(), now.getMonth(), now.getDate() - 1, 6, 0, 0, 0);
} else {
  workdayStart = new Date(now.getFullYear(), now.getMonth(), now.getDate(), 6, 0, 0, 0);
}
```

**原因**：使用 `new Date(year, month, date, hours, ...)` 构造函数直接创建本地时间，避免时区转换问题。

### 修复 2：`hourlySnapshotService.ts` - 使用本地时间提取日期

**修改前**：
```typescript
const snapshotDate = selectedTime.toISOString().split('T')[0]; // 会转换成 UTC
```

**修改后**：
```typescript
const year = selectedTime.getFullYear();
const month = String(selectedTime.getMonth() + 1).padStart(2, '0');
const day = String(selectedTime.getDate()).padStart(2, '0');
const snapshotDate = `${year}-${month}-${day}`;
```

**原因**：直接使用本地时间的年月日，不经过 UTC 转换。

### 修复 3：`TimeAxis.tsx` - "次日" 前缀显示

`formatTime` 函数已经正确实现：
```typescript
const formatTime = (date: Date): string => {
  const hours = String(date.getHours()).padStart(2, '0');
  const minutes = String(date.getMinutes()).padStart(2, '0');
  const timeStr = `${hours}:${minutes}`;
  
  // 如果是凌晨 00:00-05:59，添加"次日"前缀
  if (date.getHours() >= 0 && date.getHours() <= 5) {
    return `次日${timeStr}`;
  }
  
  return timeStr;
};
```

这个函数会在时间范围显示中使用，显示 `maxTime` 时会自动添加"次日"前缀。

## 🔧 技术要点

1. **时区处理**：始终使用本地时间，避免 UTC 转换
2. **Date 构造**：使用 `new Date(year, month, date, hours, ...)` 而不是 `setDate()` + `setHours()`
3. **日期提取**：使用 `getFullYear()` + `getMonth()` + `getDate()` 而不是 `toISOString()`
4. **时间显示**：根据小时数判断是否添加"次日"前缀
