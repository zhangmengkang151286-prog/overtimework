# 时间轴修复总结

## 🐛 问题描述

用户反馈了两个问题：

1. **时间轴显示问题**：时间轴右侧显示 "05:59"，应该显示 "次日05:59"
2. **拖动数据不变问题**：拖动时间轴时，数据不会更新

---

## ✅ 修复内容

### 1. 时间轴显示修复

**文件**：`OvertimeIndexApp/src/components/TimeAxis.tsx`

**修改**：在 `formatTime` 函数中添加"次日"前缀判断

```typescript
/**
 * 格式化时间显示 - 完整时间（小时:分钟）
 * 如果是次日凌晨（00:00-05:59），显示"次日"前缀
 */
const formatTime = (date: Date): string => {
  const hours = String(date.getHours()).padStart(2, '0');
  const minutes = String(date.getMinutes()).padStart(2, '0');
  const timeStr = `${hours}:${minutes}`;
  
  // 如果是凌晨 00:00-05:59，添加"次日"前缀
  if (date.getHours() >= 0 && date.getHours() <= 5) {
    return `次日${timeStr}`;
  }
  
  return timeStr;
};
```

**效果**：
- 06:00 → 显示 "06:00"
- 12:00 → 显示 "12:00"
- 18:00 → 显示 "18:00"
- 00:00 → 显示 "次日00:00"
- 05:59 → 显示 "次日05:59"

### 2. 拖动数据更新修复

#### 2.1 修复 `hourlySnapshotService.getSnapshot` 方法

**文件**：`OvertimeIndexApp/src/services/hourlySnapshotService.ts`

**问题**：
- 原来的方法只接受 `hour` 参数，无法处理跨日期的情况
- 查询数据库时总是使用 `today`，导致无法查询次日凌晨的快照

**修改**：
1. 将参数从 `hour: number` 改为 `selectedTime: Date`
2. 使用 `selectedTime` 的日期来查询数据库，而不是总是使用今天
3. 改进缓存逻辑，使用日期+小时作为缓存key

```typescript
/**
 * 获取指定小时的快照
 * @param selectedTime - 选择的时间（Date 对象）
 * @param currentData - 当前实时数据
 */
async getSnapshot(selectedTime: Date, currentData: RealTimeData | null): Promise<HourlySnapshot | null> {
  const now = new Date();
  const hour = selectedTime.getHours();
  
  // 如果选择的时间在当前小时内（1小时范围），返回实时数据
  const timeDiff = Math.abs(now.getTime() - selectedTime.getTime());
  if (timeDiff < 60 * 60 * 1000 && currentData) {
    return {
      hour,
      timestamp: now,
      participantCount: currentData.participantCount,
      overtimeCount: currentData.overtimeCount,
      onTimeCount: currentData.onTimeCount,
      tagDistribution: currentData.tagDistribution,
      dailyStatus: currentData.dailyStatus,
    };
  }

  // 从 Supabase 数据库读取
  // 使用选择的日期，而不是今天
  const snapshotDate = selectedTime.toISOString().split('T')[0]; // YYYY-MM-DD
  
  const {data, error} = await supabase
    .from('hourly_snapshots')
    .select('*')
    .eq('snapshot_date', snapshotDate)
    .eq('snapshot_hour', hour)
    .maybeSingle();
  
  // ... 处理数据
}
```

#### 2.2 修复 `TrendPage` 中的调用

**文件**：`OvertimeIndexApp/src/screens/TrendPage.tsx`

**修改**：将 `getSnapshot(hour, realTimeData)` 改为 `getSnapshot(selectedDate, realTimeData)`

```typescript
/**
 * 当选择的时间变化时，加载对应小时的快照
 */
useEffect(() => {
  const loadHourSnapshot = async () => {
    const selectedDate = new Date(selectedTime);
    
    console.log('[TrendPage] Loading snapshot for time:', selectedDate.toISOString());
    
    // 加载快照（传入完整的时间对象）
    const snapshot = await hourlySnapshotService.getSnapshot(selectedDate, realTimeData);
    if (snapshot) {
      setCurrentHourSnapshot({
        participantCount: snapshot.participantCount,
        overtimeCount: snapshot.overtimeCount,
        onTimeCount: snapshot.onTimeCount,
        tagDistribution: snapshot.tagDistribution,
        dailyStatus: snapshot.dailyStatus,
        timestamp: snapshot.timestamp,
      });
      console.log('[TrendPage] Snapshot loaded:', {
        time: selectedDate.toISOString(),
        participantCount: snapshot.participantCount,
        overtimeCount: snapshot.overtimeCount,
        onTimeCount: snapshot.onTimeCount,
        tagCount: snapshot.tagDistribution.length,
      });
    } else {
      // 没有快照，使用实时数据
      setCurrentHourSnapshot(null);
      console.log('[TrendPage] No snapshot available, using real-time data');
    }
  };
  
  loadHourSnapshot();
}, [selectedTime, realTimeData]);
```

## 💡 技术要点

### 1. 时间格式化
- 使用 `date.getHours()` 判断是否是凌晨（0-5点）
- 添加"次日"前缀，让用户清楚知道这是第二天的时间

### 2. 跨日期查询
- 不能总是使用 `today`，要使用 `selectedTime` 的日期
- 工作日可能跨越两个日期（今天 06:00 - 明天 05:59）
- 数据库查询需要使用正确的日期

### 3. 实时数据判断
- 使用时间差判断（1小时内）而不是只比较小时数
- 避免边界情况（比如 23:59 和 00:00）

### 4. 缓存优化
- 使用日期+小时作为缓存key
- 避免重复查询相同的快照
