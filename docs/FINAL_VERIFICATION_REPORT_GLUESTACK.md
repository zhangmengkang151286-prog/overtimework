### ⚠️ 失败的测试 (84 个)

失败的测试主要分为以下几类：

#### 1. React Native Mock 问题 (约 60 个测试)

**问题**: `TurboModuleRegistry.getEnforcing(...): 'DevMenu' could not be found`

**影响的测试文件**:
- `SetPasswordScreen.gluestack.test.tsx`
- `PasswordRecoveryScreen.gluestack.test.tsx`
- `DataManagementScreen.gluestack.test.tsx`
- 其他屏幕测试

**原因**: Jest 环境中 React Native 的 native 模块 mock 不完整

**解决方案**: 需要改进 `jest.setup.js` 中的 mock 配置

#### 2. 导航依赖问题 (约 10 个测试)

**问题**: `Cannot find module '@react-navigation/native-stack'`

**影响的测试文件**:
- `navigation.gluestack.test.tsx`

**原因**: 测试环境中缺少导航库的 mock

**解决方案**: 添加 `@react-navigation/native-stack` 的 mock

#### 3. 快照不匹配 (5 个快照)

**问题**: UI 组件快照与之前版本不匹配

**原因**: gluestack-ui 组件的渲染输出与 Tamagui 不同

**解决方案**: 运行 `npm test -- -u` 更新快照

#### 4. TypeScript 类型错误 (约 9 个测试)

**问题**: 一些测试中的类型定义不匹配

**示例**:
- `ontimeCount` vs `onTimeCount` (命名不一致)
- Redux state 类型不匹配

**解决方案**: 统一命名规范，修复类型定义
