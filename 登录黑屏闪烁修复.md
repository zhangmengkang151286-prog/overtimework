# 登录黑屏闪烁修复 - 最终版

## 🐛 问题描述

验证码登录时，点击"获取验证码"按钮会出现黑屏闪烁。

## 🔍 根本原因

**Alert.alert 弹窗导致页面失焦和黑屏**
- `Alert.alert` 是系统原生弹窗，会让应用失去焦点
- 在某些设备上触发背景色重绘，造成明显的黑屏闪烁
- 即使延迟显示也无法完全避免这个问题

## ✅ 最终解决方案

### 完全移除成功时的 Alert 弹窗

不再使用 `Alert.alert` 显示"验证码已发送"的提示，改为：

1. **倒计时反馈**：按钮显示"60秒"倒计时
2. **UI 文本提示**：在输入框下方显示绿色成功提示
3. **按钮状态变化**：按钮变为禁用状态

```typescript
// 修复前 - 使用 Alert 导致黑屏
if (result.success) {
  setCountdown(60);
  Alert.alert('成功', '验证码已发送到您的手机，请查收'); // ❌ 导致黑屏
}

// 修复后 - 使用 UI 反馈
if (result.success) {
  setCountdown(60);
  setSmsSent(true); // ✅ 设置成功标记
  setLoading(false);
  // 通过 UI 显示成功提示，不使用 Alert
}
```

### UI 反馈实现

在验证码输入框下方显示动态提示：

```typescript
<Box mt="$2" minHeight={40}>
  {smsSent && countdown > 0 ? (
    <Text color="$textDark50" size="sm">
      ✓ 验证码已发送，请查收短信
    </Text>
  ) : (
    <Text color="$textDark400" size="sm">
      新用户将进入注册流程
    </Text>
  )}
</Box>
```

### 失败时仍使用 Alert

只在失败时才显示 Alert（失败频率低，影响小）：

```typescript
} else {
  setLoading(false);
  setTimeout(() => {
    Alert.alert('发送失败', result.error || '验证码发送失败');
  }, 100);
}
```

## 📝 修改的代码

### 1. 添加状态标记

```typescript
const [smsSent, setSmsSent] = useState(false); // 验证码发送成功标记
```

### 2. 修改发送逻辑

```typescript
const handleSendSMSCode = async () => {
  if (!validatePhoneNumber(phoneNumber)) {
    return;
  }

  try {
    setLoading(true);
    setSmsSent(false); // 重置状态
    const result = await AuthService.sendSMSCode(phoneNumber, 'login');

    if (result.success) {
      setCountdown(60);
      setSmsSent(true); // ✅ 设置成功标记
      setLoading(false);
      // 不使用 Alert，通过 UI 反馈
    } else {
      setLoading(false);
      setTimeout(() => {
        Alert.alert('发送失败', result.error || '验证码发送失败');
      }, 100);
    }
  } catch (error: any) {
    setLoading(false);
    setTimeout(() => {
      Alert.alert('发送失败', error.message || '验证码发送失败');
    }, 100);
  }
};
```

### 3. 添加 UI 反馈

```typescript
<Box mt="$2" minHeight={40}>
  {smsSent && countdown > 0 ? (
    <Text color="$textDark50" size="sm">
      ✓ 验证码已发送，请查收短信
    </Text>
  ) : (
    <Text color="$textDark400" size="sm">
      新用户将进入注册流程
    </Text>
  )}
</Box>
```

## 💡 设计理念

### 为什么不用 Alert？

1. **系统原生弹窗的问题**
   - Alert 会让应用失去焦点
   - 触发系统级别的 UI 重绘
   - 在某些设备上会导致黑屏闪烁

2. **更好的用户体验**
   - 内联提示不打断用户操作
   - 视觉反馈更自然流畅
   - 符合现代 APP 设计规范

3. **多重反馈机制**
   - 倒计时：明确告知等待时间
   - 文本提示：确认操作成功
   - 按钮禁用：防止重复点击

### 什么时候可以用 Alert？

- 错误提示（频率低）
- 需要用户确认的操作
- 重要的警告信息
