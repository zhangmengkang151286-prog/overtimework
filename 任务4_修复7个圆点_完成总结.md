## 问题描述

用户报告：7个历史状态圆点显示数据都是0（除了1月30日有32人的数据）

**日期**: 2026年2月5日（星期四）

## 根本原因

**没有自动归档服务在运行**

- `get_daily_status()` 函数从 `daily_history` 表读取数据
- `daily_history` 表需要每天从 `status_records` 表归档数据
- 目前没有任何自动化任务在执行归档操作
- 导致除了手动归档的1月30日，其他日期都没有数据

## 技术细节

### 数据流程

```
status_records (实时提交)
    ↓
每天 06:00 北京时间
    ↓
GitHub Actions 触发
    ↓
调用 archive_daily_data() 函数
    ↓
归档到 daily_history 表
    ↓
get_daily_status() 函数读取
    ↓
应用显示7个圆点
```

### 归档函数逻辑

```sql
archive_daily_data(target_date)
  ↓
1. 从 status_records 读取指定日期的数据
2. 计算统计数据：
   - participant_count: COUNT(DISTINCT user_id)
   - overtime_count: SUM(CASE WHEN is_overtime THEN 1 ELSE 0 END)
   - on_time_count: SUM(CASE WHEN NOT is_overtime THEN 1 ELSE 0 END)
   - tag_distribution: 标签分布 (JSONB)
3. 插入或更新 daily_history 表
4. 返回归档结果
```

### 7个圆点显示逻辑

```typescript
// HistoricalStatusIndicator.tsx
getStatusColor(status) {
  switch (status) {
    case 'overtime':  // 加班人数 > 准点人数
      return '#FFB3B3'; // 浅红色
    case 'ontime':    // 准点人数 >= 加班人数
      return '#B3FFB3'; // 浅绿色
    case 'pending':   // 今天（未确定）
      return '#FFEB99'; // 浅黄色（闪烁）
  }
}
```

## 监控和维护

### 定期检查（建议每周一次）

```sql
-- 检查最近7天是否都有归档数据
WITH date_series AS (
  SELECT generate_series(
    CURRENT_DATE - INTERVAL '6 days',
    CURRENT_DATE - INTERVAL '1 day',
    '1 day'::interval
  )::DATE as date
)
SELECT 
  ds.date as "日期",
  CASE 
    WHEN dh.date IS NULL THEN '❌ 缺失'
    ELSE '✅ 存在'
  END as "状态",
  COALESCE(dh.participant_count, 0) as "参与人数"
FROM date_series ds
LEFT JOIN daily_history dh ON ds.date = dh.date
ORDER BY ds.date DESC;
```

## 故障排除

### 如果圆点还是显示0

1. **检查 daily_history 表**
   ```sql
   SELECT * FROM daily_history ORDER BY date DESC LIMIT 7;
   ```
   - 如果没有数据，执行 `manual_archive_recent_days.sql`

2. **检查 get_daily_status 函数**
   ```sql
   SELECT * FROM get_daily_status(7);
   ```
   - 如果返回空或错误，检查函数定义

3. **重新加载应用**
   - 手机上摇一摇 → Reload
   - 查看 Metro 控制台日志

### 如果 GitHub Actions 失败

1. **检查 Secrets 配置**

   - `SUPABASE_ANON_KEY`

2. **查看 Actions 日志**
   - 找到具体的错误信息
   - 根据错误信息调整

3. **手动执行归档**
