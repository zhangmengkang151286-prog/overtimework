# 工作日重置功能实现计划

## 功能概述

实现"工作日"概念（06:00-次日05:59），并添加进度条倒计时功能。

## 实现步骤

### 1. 数据库层面修改

#### 1.1 创建工作日计算函数
- 文件：`OvertimeIndexApp/implement_workday_logic.sql`
- 功能：计算当前时间对应的工作日日期
- 逻辑：凌晨 0:00-5:59 算前一天

#### 1.2 更新统计函数
- 修改 `get_real_time_stats()` 使用工作日逻辑
- 修改 `get_top_tags()` 使用工作日逻辑
- 修改物化视图使用工作日逻辑

#### 1.3 更新归档函数
- 修改 `archive_daily_data()` 在早上6:00执行
- 更新定时任务为 `0 6 * * *`（UTC）或 `0 22 * * *`（北京时间）

### 2. 前端UI修改

#### 2.1 创建倒计时Hook
- 文件：`OvertimeIndexApp/src/hooks/useWorkdayCountdown.ts`
- 功能：
  - 计算距离下次重置的剩余时间
  - 计算进度百分比
  - 返回格式化的倒计时文本
  - 根据剩余时间返回颜色状态

#### 2.2 修改TrendPage
- 文件：`OvertimeIndexApp/src/screens/TrendPage.tsx`
- 修改：
  - 更新"今日参与记录人数"为"本轮参与人数"
  - 添加进度条组件
  - 添加倒计时显示
  - 每分钟更新倒计时

### 3. 文档更新

#### 3.1 更新归档设置指南
- 文件：`OvertimeIndexApp/DAILY_ARCHIVE_SETUP.md`
- 更新为工作日逻辑说明

#### 3.2 创建用户说明
- 说明工作日概念
- 说明重置时间

## UI设计

```
┌─────────────────────────────┐
│  本轮参与人数                │
│  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░  78%    │  ← 进度条（动态颜色）
│  距离重置 5小时23分          │  ← 倒计时
│                             │
│      5,234                  │  ← 大数字
└─────────────────────────────┘
```

### 进度条颜色规则
- 剩余 > 6小时：绿色 (#4ade80)
- 剩余 3-6小时：黄色 (#fbbf24)
- 剩余 1-3小时：橙色 (#fb923c)
- 剩余 < 1小时：红色 (#ef4444)

### 倒计时文本规则
- 剩余 > 6小时：显示"距离重置 X小时"
- 剩余 1-6小时：显示"距离重置 X小时Y分"
- 剩余 < 1小时：显示"距离重置 X分钟"
- 剩余 < 5分钟：显示"即将重置"（闪烁）

## 注意事项

1. **时区问题**：Supabase使用UTC，需要考虑时区转换
2. **向后兼容**：确保现有数据不受影响
3. **性能优化**：倒计时每分钟更新一次即可
4. **用户体验**：进度条和倒计时要平滑更新
