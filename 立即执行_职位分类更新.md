## 🔍 数据验证

执行脚本后，在 Supabase SQL Editor 中运行以下查询验证：

```sql
-- 查看所有行业分类和职位数量
SELECT 
  subcategory AS "行业分类",
  COUNT(*) AS "职位数量"
FROM tags
WHERE type = 'position' AND is_active = true
GROUP BY subcategory
ORDER BY subcategory;

-- 查看职位总数
SELECT COUNT(*) AS "职位总数" 
FROM tags 
WHERE type = 'position' AND is_active = true;

-- 查看某个行业的具体职位
SELECT name, subcategory
FROM tags
WHERE type = 'position' 
  AND subcategory = '互联网、软件与信息技术'
  AND is_active = true
ORDER BY name;

-- 查看新经济相关职位
SELECT name, subcategory
FROM tags
WHERE type = 'position' 
  AND subcategory IN ('现代物流、快递与即时配送', '电子商务与直播经济')
  AND is_active = true
ORDER BY subcategory, name;
```

## 🐛 常见问题

### 问题1: 职位不显示分类

**原因**：缓存未清除

**解决**：
```bash
cd OvertimeIndexApp
npx expo start --clear
```

### 问题2: 数据库脚本执行失败

**原因**：可能是权限问题或表结构不匹配

**解决**：
1. 检查 Supabase 项目是否正确
2. 确认 `tags` 表存在
3. 查看错误信息，根据提示调整

### 问题3: 职位数量不对

**原因**：可能有重复数据或部分数据未插入

**解决**：
```sql
-- 清空职位数据重新执行
DELETE FROM tags WHERE type = 'position';
-- 然后重新执行完整的 SQL 脚本
```
