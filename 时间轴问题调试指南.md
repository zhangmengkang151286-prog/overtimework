## 🔍 调试步骤

### 步骤1：检查代码是否正确

#### 1.1 检查 TimeAxis.tsx

打开 `OvertimeIndexApp/src/components/TimeAxis.tsx`，找到 `formatTime` 函数：

```typescript
const formatTime = (date: Date): string => {
  const hours = String(date.getHours()).padStart(2, '0');
  const minutes = String(date.getMinutes()).padStart(2, '0');
  const timeStr = `${hours}:${minutes}`;
  
  // 如果是凌晨 00:00-05:59，添加"次日"前缀
  if (date.getHours() >= 0 && date.getHours() <= 5) {
    return `次日${timeStr}`;
  }
  
  return timeStr;
};
```

**验证**：这个函数应该在 170-182 行左右。

#### 1.2 检查 hourlySnapshotService.ts

打开 `OvertimeIndexApp/src/services/hourlySnapshotService.ts`，找到 `getSnapshot` 方法：

```typescript

    // ...
  }
  
  // 使用选择的日期，而不是今天

  const {data, error} = await supabase
```

**验证**：方法签名应该是 `getSnapshot(selectedTime: Date, currentData: RealTimeData | null)`

## 🔧 常见问题

### Q1: 时间轴显示还是 "05:59"

**原因**：代码没有生效或 App 没有重启

**解决**：
1. 确认 `formatTime` 函数已修改
2. 重启 App 并清除缓存
3. 检查是否有 TypeScript 编译错误

### Q2: 拖动时数据不变

**原因1**：数据库没有快照数据

**解决**：
```sql
-- 执行此脚本重新生成快照
-- OvertimeIndexApp/fix_timezone_final.sql
```

**原因2**：`getSnapshot` 方法参数错误

**解决**：
- 检查是否传入 `selectedTime: Date` 而不是 `hour: number`
- 检查是否使用 `selectedTime` 的日期查询数据库

**原因3**：缓存问题

**解决**：
```typescript
// 在 hourlySnapshotService.ts 中，清除缓存
clearAllSnapshots() {
  this.snapshots.clear();
  console.log('[HourlySnapshot] All snapshots cleared');
}
```

### Q3: 控制台没有日志

**原因**：`useEffect` 没有触发

**解决**：
1. 检查 `selectedTime` 是否正确更新
2. 检查依赖数组 `[selectedTime, realTimeData]`
3. 添加更多日志：

```typescript
useEffect(() => {
  console.log('[TrendPage] selectedTime changed:', selectedTime);
  const loadHourSnapshot = async () => {
    // ...
  };
  loadHourSnapshot();
}, [selectedTime, realTimeData]);
```

### Q4: 数据库查询返回 null

**原因**：快照数据不存在

**解决**：
1. 检查数据库中是否有快照数据
2. 执行 `fix_timezone_final.sql` 生成快照
3. 检查查询条件是否正确：

```sql
SELECT * FROM hourly_snapshots
WHERE snapshot_date = '2026-01-31'
AND snapshot_hour = 7;
```
