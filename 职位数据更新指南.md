# 职位数据更新指南

## 问题说明

当前应用中的职位选择器显示的是旧的职位列表，需要更新为新的 300 个职位。

## 原因分析

应用中的职位数据是从 Supabase 数据库的 `positions` 表中读取的，而不是直接使用前端代码中的数据。因此需要更新数据库中的职位数据。

## 更新步骤

### 1. 在 Supabase SQL Editor 中执行更新脚本

1. 登录 Supabase Dashboard
2. 进入你的项目
3. 点击左侧菜单的 "SQL Editor"
4. 创建新查询
5. 复制 `update_positions_complete.sql` 文件的内容
6. 粘贴到 SQL Editor 中
7. 点击 "Run" 执行

### 2. 验证更新结果

执行完成后，你会看到两个查询结果：

1. 按行业分类的职位数量统计
2. 职位总数（应该显示约 300 个）

### 3. 重启应用

更新数据库后，需要重启应用以清除缓存：

```bash
# 清除缓存并重启
npx expo start --clear
```

## 注意事项

1. **备份数据**：执行脚本前建议先备份数据库
2. **TRUNCATE 操作**：脚本会清空现有的 positions 表数据，请确认无重要数据
3. **级联删除**：使用了 CASCADE 选项，会自动处理外键关联
4. **用户数据不受影响**：用户已选择的职位信息存储在 user_profiles 表中，不会被删除

## 如果遇到问题

### 问题1：执行脚本时报错

**可能原因**：positions 表不存在或结构不匹配

**解决方案**：
```sql
-- 检查表结构
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'positions';
```

### 问题2：更新后应用仍显示旧数据

**可能原因**：缓存未清除

**解决方案**：
```bash
# 完全清除缓存
npx expo start --clear

# 如果还不行，删除 node_modules 重新安装
rm -rf node_modules
npm install
npx expo start --clear
```

### 问题3：职位选择器为空

**可能原因**：数据库查询失败或 RLS 策略限制

**解决方案**：
```sql
-- 检查数据是否插入成功
SELECT COUNT(*) FROM positions WHERE is_active = true;

-- 检查 RLS 策略
SELECT * FROM positions LIMIT 10;
```
