## 🚀 使用步骤

### 步骤 1: 在 Supabase 创建测试用户

在 Supabase SQL Editor 中执行 `create_test_user.sql`:

```bash
cd OvertimeIndexApp
# 打开 create_test_user.sql 文件
# 复制内容到 Supabase Dashboard > SQL Editor
# 点击 Run 执行
```

### 步骤 2: 重启应用

```bash
cd OvertimeIndexApp
npx expo start --tunnel --clear
```

### 步骤 3: 登录测试账号

1. 应用启动后会显示登录界面
2. 点击 "🧪 测试账号登录" 按钮
3. 看到"登录成功"提示后点击"确定"
4. 进入主界面

## ✨ 功能测试

## 📊 测试场景

### 场景 1: 准时下班

1. 登录测试账号
2. 点击"提交今日状态"
3. 选择"准点下班"
4. 选择标签"任务完成"

4. 选择标签"项目紧急"
5. 选择加班时长"2小时"
6. 点击"确认提交"
7. 确认提交成功
8. 检查 Supabase 数据库

## 🔄 重置测试数据

如果需要重新测试，可以删除今天的提交记录：

```sql
-- 删除测试用户今天的状态记录
DELETE FROM status_records 
WHERE user_id = 'test-user-001' 
AND date = CURRENT_DATE;
```

或者清除应用本地存储：

```bash
# 在应用中
# 设置 > 数据管理 > 清除本地数据
```

## ⚠️ 注意事项

1. **测试账号是共享的**：如果多人使用同一个测试账号，数据会互相覆盖
2. **每天只能提交一次**：如果已经提交过，需要删除记录或等到第二天
3. **数据会保存到 Supabase**：测试数据会真实保存到数据库中

## 🐛 常见问题

### Q: 点击登录按钮没有反应？
A: 检查控制台日志，可能是网络问题或 Supabase 连接问题

### Q: 登录成功但没有跳转？
A: 检查 Redux store 是否正确更新了用户信息

### Q: 提交状态失败？
A: 检查：
- 网络连接
- Supabase 配置
- 数据库中是否存在测试用户
- 是否已经提交过今天的状态

### Q: 如何退出登录？
A: 目前需要清除应用数据或重新安装应用
