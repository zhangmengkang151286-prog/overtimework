# Task 11: VersusBar 组件手动测试指南

## 测试目的

验证 VersusBar 组件从 Tamagui 迁移到 gluestack-ui 后功能正常。

## 测试环境

- React Native + Expo
- gluestack-ui v1.x
- react-native-reanimated

## 测试步骤

### 方法 1: 在应用中直接测试（推荐）

VersusBar 组件已在 TrendPage 中使用，可以直接运行应用查看效果。

```bash
# 1. 启动应用
npx expo start

# 2. 在手机或模拟器上打开应用

# 3. 导航到趋势页面（TrendPage）

# 4. 观察 VersusBar 组件的显示效果
```

### 方法 2: 使用验证文件测试

```bash
# 1. 备份当前的 App.tsx
cp App.tsx App.tsx.backup

# 2. 将 verify-versus-bar.tsx 的内容复制到 App.tsx
# 或者直接重命名
mv App.tsx App.tsx.old
mv verify-versus-bar.tsx App.tsx

# 3. 启动应用
npx expo start

# 4. 在手机或模拟器上打开应用

# 5. 测试完成后恢复 App.tsx
mv App.tsx.old App.tsx
```

## 预期结果

### 正常情况

1. **进度条显示**：
   - 左侧红色部分表示加班人数比例
   - 右侧绿色部分表示准时人数比例
   - 两部分宽度之和等于容器宽度

2. **标签显示**：
   - 左侧显示"加班 X"（红色）
   - 右侧显示"准时 Y"（绿色）
   - 数字与传入的 props 一致

3. **动画效果**：
   - 数据变化时进度条平滑过渡
   - 无卡顿或闪烁

4. **深色模式**：
   - 自动适配深色主题
   - 颜色对比度良好

### 异常情况

如果出现以下问题，请报告：

1. **进度条不显示**：
   - 检查 gluestack-ui 是否正确安装
   - 检查 GluestackUIProvider 是否正确配置

2. **颜色不正确**：
   - 检查是否使用了正确的 tokens
   - 检查深色模式配置

3. **动画卡顿**：
   - 检查 react-native-reanimated 是否正确安装
   - 检查设备性能

4. **类型错误**：
   - 检查 TypeScript 配置
   - 检查组件 props 类型

## 对比测试

### 迁移前（Tamagui 版本）

- 使用自定义颜色（#00D9FF, #EF4444）
- 需要手动传入 theme 属性
- 使用 StyleSheet

### 迁移后（gluestack-ui 版本）

- 使用 gluestack-ui tokens（$error500, $success500）
- 自动适配主题（无需 theme 属性）
- 使用 gluestack-ui 样式系统
