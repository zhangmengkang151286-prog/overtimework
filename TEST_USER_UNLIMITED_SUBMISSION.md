# 测试用户无限提交功能

## 功能说明

### 普通用户
- ✅ 每天只能提交一次状态
- ✅ 提交后，"✍️ 提交今日状态" 按钮消失
- ✅ 显示 "✅ 今日状态已提交" 提示
- ✅ 长按提示 2 秒可重置状态（用于调试）

### 测试用户（ID: `00000000-0000-0000-0000-000000000001`）
- ✅ 可以无限次提交状态
- ✅ "✍️ 提交今日状态" 按钮始终显示
- ✅ 按钮下方显示 "(测试账号 - 可多次提交)" 提示
- ✅ 不显示 "今日状态已提交" 提示
- ✅ 方便测试各种场景

## 实现逻辑

### 1. 用户识别

```typescript
// src/hooks/useUserStatus.ts
const isTestUser = useCallback(() => {
  return currentUser?.id === '00000000-0000-0000-0000-000000000001';
}, [currentUser]);
```

```typescript
// 测试用户始终显示提交按钮，普通用户根据是否已提交决定
```

## 使用场景

### 测试用户的使用场景

1. **测试准时下班流程**
   - 提交准时下班状态
   - 选择不同的标签
   - 验证数据是否正确保存

2. **测试加班流程**
   - 提交加班状态
   - 输入不同的加班时长
   - 选择不同的加班原因

3. **测试多次提交**
   - 连续提交多次
   - 验证数据累积是否正确
   - 测试数据可视化更新

4. **测试边界情况**
   - 提交空标签
   - 提交极大加班时长
   - 快速连续提交

### 普通用户的使用场景

1. **每日提交**
   - 每天只能提交一次
   - 提交后按钮消失
   - 第二天自动重置

2. **误操作重置**
   - 长按 "今日状态已提交" 2 秒
   - 确认重置
   - 重新提交

## 注意事项

### 测试用户 ID

测试用户的 UUID 是固定的：
```
00000000-0000-0000-0000-000000000001
```

如果需要添加更多测试用户，可以在代码中添加：

```typescript
const isTestUser = useCallback(() => {
  const testUserIds = [
    '00000000-0000-0000-0000-000000000001',
    '00000000-0000-0000-0000-000000000002', // 第二个测试用户
  ];
  return testUserIds.includes(currentUser?.id || '');
}, [currentUser]);
```

### 数据库记录

测试用户的多次提交会在数据库中创建多条记录。如果需要清理：

```sql
-- 删除测试用户的所有记录
DELETE FROM status_records 
WHERE user_id = '00000000-0000-0000-0000-000000000001';
```

### 生产环境

在生产环境中，测试用户仍然可以无限提交。如果不希望这样：

1. 在生产环境中删除测试用户
2. 或者添加环境检查：

```typescript
const isTestUser = useCallback(() => {
  // 仅在开发环境允许测试用户无限提交
  if (!__DEV__) return false;
  return currentUser?.id === '00000000-0000-0000-0000-000000000001';
}, [currentUser]);
```
