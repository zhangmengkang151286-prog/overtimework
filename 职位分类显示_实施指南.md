### 步骤3: 更新服务层映射

`supabaseService.ts` 中的 `getTags` 方法需要从 `positions` 表读取数据。

检查 `mapDatabaseTagToTag` 方法是否正确映射 `category` 到 `subcategory`:

```typescript
private mapDatabaseTagToTag(
  dbTag: Database['public']['Tables']['tags']['Row'],
): Tag {
  return {
    id: dbTag.id,
    name: dbTag.name,
    type: dbTag.type,
    category: (dbTag as any).category || 'ontime',
    subcategory: (dbTag as any).subcategory || undefined, // 这里需要映射 category
    isActive: dbTag.is_active,
    usageCount: dbTag.usage_count,
    createdAt: new Date(dbTag.created_at),
  };
}
```

**注意**: 如果 `positions` 表使用 `category` 字段存储行业分类,需要将其映射到 `subcategory`,因为 `SearchableSelector` 组件使用 `subcategory` 进行分组。

### 步骤4: 确认数据表结构

检查你的应用是从哪个表读取职位数据:

**选项A: 使用 `tags` 表**
- 如果职位存储在 `tags` 表中,需要确保 `type='position'` 的记录有 `subcategory` 字段

**选项B: 使用 `positions` 表**
- 如果有独立的 `positions` 表,需要修改 `getTags` 方法支持从 `positions` 表读取
- 或者创建一个新的 `getPositions` 方法

### 步骤5: 修改服务层(如果需要)

如果使用独立的 `positions` 表,在 `supabaseService.ts` 中添加:

```typescript
/**
 * 获取职位列表(从 positions 表)
 */
async getPositions(search?: string): Promise<Tag[]> {
  try {
    let query = supabase
      .from('positions')
      .select('*')
      .eq('is_active', true);

    if (search) {
      query = query.ilike('name', `%${search}%`);
    }

    query = query.order('usage_count', {ascending: false});

    const {data, error} = await query;

    if (error) throw handleSupabaseError(error);

    // 将 positions 表数据映射为 Tag 格式
    return (data || []).map(pos => ({
      id: pos.id,
      name: pos.name,
      type: 'position',
      category: 'position',
      subcategory: pos.category, // 将 category 映射到 subcategory 用于分组
      isActive: pos.is_active,
      usageCount: pos.usage_count || 0,
      createdAt: new Date(pos.created_at),
    }));
  } catch (error) {
    console.error('Get positions error:', error);
    throw error;
  }
}
```

## 常见问题

### 问题1: 职位不显示分类

**原因**: `subcategory` 字段为空

**解决**: 确保数据库中 `category` 字段有值,并且正确映射到 `subcategory`

### 问题2: 分类顺序不对

**原因**: `SearchableSelector` 按 `subcategory` 首次出现顺序排序

**解决**: 可以在 SQL 中调整插入顺序,或修改组件排序逻辑

### 问题3: 搜索后不显示分类

**原因**: 搜索模式下会扁平显示结果

**解决**: 这是正常行为,搜索时不需要分类
