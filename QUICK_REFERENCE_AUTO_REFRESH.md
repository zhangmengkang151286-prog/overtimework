✅ **参与人数 + 加班/准点对比** → 每 **3秒** 自动刷新

```
排名 | 标签名称 | 使用次数 | 占比
-----|---------|---------|------
  1  | 项目紧急 |   45    | 15.2%
```

## 测试步骤

### 1. 启动应用
```bash
cd OvertimeIndexApp
npm start
```

### 2. 观察自动刷新

**参与人数区域**（3秒刷新）：
- 打开控制台，查看日志：`Auto-refreshing stats (3s interval)...`
- 观察参与人数是否每3秒变化
- 观察加班/准点对比条是否每3秒更新

**标签分布区域**（7秒刷新）：
- 打开控制台，查看日志：`Auto-refreshing tag data (7s interval)...`
- 观察标签分布图是否每7秒变化
- 观察标签占比列表是否每7秒更新

### 3. 提交状态测试

1. 点击"✍️ 提交今日状态"
2. 选择加班或准点
3. 选择标签
4. 提交后观察：
   - 参与人数应该增加（3秒内更新）
   - 标签分布应该变化（7秒内更新）
   - 标签占比列表应该更新（7秒内更新）

### 4. 网络测试

**断网测试**：
1. 关闭网络连接
2. 自动刷新应停止
3. 显示"⚠️ 网络连接已断开"提示

**恢复网络**：
1. 恢复网络连接
2. 自动刷新应立即恢复
3. 数据应该更新到最新

## 常见问题

### Q: 数据没有自动刷新？
A: 检查：
1. 网络是否连接
2. 控制台是否有错误日志
3. 是否在查看历史数据（历史模式不自动刷新）

### Q: 刷新太频繁/太慢？
A: 当前设置：
- 统计数据：3秒
- 标签数据：7秒
- 可以在 `TrendPage.tsx` 中修改 `setInterval` 的时间参数

### Q: 标签占比列表不显示？
A: 检查：
1. 是否有标签数据
2. 是否有用户提交过状态
3. 控制台是否有错误

### Q: 如何暂停自动刷新？
A: 当前版本自动刷新始终启用（网络连接时）
- 未来版本将添加手动控制开关
